# 完整 Proposal 模板 v1.1

## 1. 标题与一句话 story

“我们首次 [做了 X]，通过 [方法 Y]，证明了 [发现 Z]，这对 [领域 W] 意味着 [影响]。”

若经历重定位，story 必须直面竞品：

“X 类方法做不到 A，我们的 B 机制给出 A。”

## 2. 研究问题

- 精确假设：可证伪陈述。
- 为什么现在做：新数据 / 新工具 / 新争论 / 竞品能力缺口 / 时效窗口。
- 不做什么：明确边界，避免过度 claim。

## 3. 相关工作定位

- 坐标系图：两个关键维度上画出邻近工作与本工作的空位。
- 与最近似 2-3 篇工作（含 L3 查新发现竞品）的逐点差异表。
- 最强 baseline 清单：必须包含查新发现的最强竞品。

## 4. 技术路线

- 主路线：分阶段，每阶段的输入 / 输出 / 验证方式。
- 最大风险点识别 + Plan B。
- 创新点清单：每条对应一个消融实验。
- 失败预测：什么现象说明假设不成立？

## 5. 条件匹配度分析

| 需求项 | 现有条件 | 状态（够用/缺口/可替代） | 补齐方案 |
|--------|---------|--------------------------|---------|
| 数据 | | | |
| 算力（估算 X 卡 × Y 天） | | | |
| 代码 / baseline | | | |
| 技能 | | | |
| 时间 | | | |
| 经费 / API / 标注 | | | |
| 合规许可 | | | |

## 6. 实验设计

- 数据集：主实验 + 泛化验证 + 压力测试。
- baseline：必须包含最强竞品、trivial baseline、用户方法的简化版本。
- 指标：主指标 + 辅助指标 + 真实场景指标。
- 消融计划：每个创新点的隔离验证。
- 主攻实验：在竞品弱区设计决定性对比。
- 统计严谨性：随机种子数、置信区间、显著性检验方式。
- 最小有意义提升：按领域指标定义，不用固定 1%。

## 7. 伦理合规与数据边界

- 是否涉及隐私 / 生物识别 / 医疗 / 监控 / 未成年人 / 敏感属性。
- 数据来源是否授权。
- 是否需要匿名化、脱敏、IRB、数据使用协议。
- 风险是否可通过任务重定义、场景限制、替代数据或输出限制降低。

## 8. 结果预案

- 预期结果及解读。
- 结果相反 / 不显著时：能否转化为发现？
- 最小可发表单元是什么？
- 哪些负结果应该及时止损？

## 9. 发表规划

- 主投 + 备选阵地及 story 适配理由。
- 时效窗口“紧张”时：preprint / 短文 / workshop / tech report 占位节点。
- 若目标为顶刊：说明从短文到长文需要补哪些体系化实验。

## 10. 时间线

里程碑粒度：

```text
M0 查新锁定 + baseline 复现
M1 可行性验证
M2 主模型完成
M3 主实验完成
M4 消融 + 泛化 + 失败分析
M5 写作 + 图表 + rebuttal 预案
M6 投稿 / preprint
```

时间线必须受时效窗口硬约束；窗口紧张时压缩到最小可发表单元。
